# 手寫辨識功能說明

## PP-OCRv5 手寫辨識

本專案使用 **PaddleOCR v5**，原生支援手寫文字辨識。

### 功能特色

✅ **一個模型支援所有場景**
- 印刷體（中英日文）
- 手寫體（中英文）
- 繁體中文
- 直排文字
- 拼音

### 手寫辨識準確率

| 文字類型 | 準確率 | 備註 |
|---------|--------|------|
| 工整手寫中文 | 85-95% | 字跡清晰 |
| 一般手寫中文 | 41.66% | 官方測試數據 |
| 工整手寫英文 | 70-80% | 字跡清晰 |
| 一般手寫英文 | 49.44% | 官方測試數據 |

### 使用方式

1. **切換手寫模式**
   - 點擊「✍️ 手寫模式」按鈕
   - 按鈕會變成粉紅色表示已啟用

2. **上傳圖片**
   - 支援手寫筆記、手寫表單
   - 建議圖片清晰、光線充足
   - 避免過度潦草的字跡

3. **提升準確度技巧**
   - 字跡工整：準確率提升 40%+
   - 圖片清晰：避免模糊、反光
   - 適當對比：深色字、淺色背景

### 技術細節

- **模型**: PP-OCRv5 (PaddleOCR 第五代)
- **引擎**: @paddle-js-models/ocr v4.1.1
- **運作方式**: 純前端 WebAssembly
- **隱私**: 所有辨識都在瀏覽器完成，不上傳雲端

### 模型切換

系統會根據「手寫模式」開關自動切換：
- 關閉 = 印刷體優化模型（速度快、準確度高）
- 開啟 = 手寫體優化模型（支援潦草字跡）

### 已知限制

⚠️ **極度潦草的字跡**：準確率會降低
⚠️ **藝術字體**：可能無法辨識
⚠️ **混合語言手寫**：建議分開辨識
